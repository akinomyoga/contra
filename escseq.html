制御シーケンス 一般論

制御シーケンス (エスケープシーケンス) とは、 端末などでの文字列の交換(やりとり)に於いて、 通常の文字列の中に埋め込まれる「特別な情報を符号化したバイト列」の事である。 制御シーケンスは、

  • "スペース・タブ・改行・改頁・カーソル移動" など文字列の配置に関する制御
  • "通信の開始・終了" や "文字コードの切替" などの通信自体に関する制御
  • "文字の色・字体・背景色・下線・記号" などの文字列の修飾に関する制御
  • "カーソルキー・Functionキー・テンキー・マウス操作" などの特別な入力に関する制御
  • "文字の表示領域のサイズ・ウィンドウのタイトルの通知・変更" などの制御
  • その他、様々な特別な機能の制御

等に用いられる。 また、これらの制御シーケンスは入力 (端末から送られるバイト列) の中で使われる場合と、 出力 (プログラムから吐き出されるバイト列) の中で使われる場合で意味が異なる場合がある事に注意する必要がある。 また、同じシーケンスであっても端末によって対応していなかったり、異なる機能が割り当てられていたりする事にも注意する。

  • C0, C1, GL 等とした物は、バイト値のグループ分けの事で、 [0x00, 0x20) の範囲の文字の集合を C0 (ASCII control characters) と呼び、 [0x80, 0xA0) の範囲の文字の集合を C1 (8-bit control characters) と呼ぶ。 この C0 C1 の二つの文字の集合を総称して制御文字 (control character, 制御キャラクタ, 制御機能文字) と呼ぶ。 一方で、[0x20,0x80) の範囲のバイトの集合を GL と呼び、 [0xA0,0xFF] の範囲のバイトの集合を GR (8-bit graphic code) と呼ぶ。 GL のバイトには通常、キーボードの表面に印刷されている様な普通の文字 (ASCII文字) が割り当てられている。 GR のバイトには文字集合 (character set) に応じて様々な文字が割り当てられる。 GL GR に割り当てられる文字を総称して図形文字 (graphic character) と呼ぶ。 但し、特に GL に属する SP 及び DEL については、 制御文字的な性格が多少あるので上の表の隅に載せた。
  • 制御文字の中でも、或る物は単独で一つの意味を持ち、 或る物は後続するバイト列と一連の系列を為して一つ意味を持つ。 特に、単独で意味を為す制御文字を制御コード (control code) と呼び、 複数バイトで一つの意味を為す系列を制御シーケンス (control sequence, 制御系列) または エスケープシーケンス (escape sequence) と呼ぶ。 表中で赤色で示した物が、制御シーケンスの先頭になる制御文字である。
  • 列の項目について

    "名称" の列には ASCII や、各端末に於いて定義された機能の短縮名を示してある。 "16進"　の列は 16 進数による C リテラル表現、 "8進" の列は 8 進数による C リテラル表現でバイト値を示す。 "R/C" の列は ISO/IEC 2022 で使用される様な、行と列によるバイト表記である。 "CN" の列は caret notation によるバイトの表現である。 キーボード上で Control+A の入力は 0x1F&'A' の様に、 但し、C1 の文字に対応する caret notation はないので、 ここだけの表現として、バイトの Meta 修飾 (c|=0x80) を ⌥ を先頭に冠して表現する事にした。 "ES" の列には、様々の言語の文字リテラル中で用いられる特別な表現があれば、それを記した。

  • ISO/IEC 2022 に従うと、
    • 2バイト目が範囲 [\x30, \x40) にあるシーケンスは私用の為に保留されている。
    • 2バイト目が範囲 [\x20, \x30) にあるシーケンスは、 主に ISO/IEC 2022 に基づく文字集合設定の為のエスケープシーケンスである。 但し、この組合せで始まるシーケンスの内でも、 [\x30, \x40) の文字で終端する物は私用の為に保留されている。
    • 2バイト目が範囲 [\x40, \x60) にあるシーケンスは、C1 の文字を 7bit で符号化するのに用いられる。 しかし、全ての C1 領域の符号が現代に於いて使われているとは限らず、 実際の端末・端末エミュレータにおいては、 歴史的な背景から、この領域の特定の文字に別の機能を割り当てている事がある。
    • 2バイト目が範囲 [\x60, \x80) にあるシーケンス及び SS2, SS3 は、 主に ISO/IEC 2022 に基づく文字集合設定の為の制御である。
    • -R- とした物は現在は規定されていないが、将来の規格変更・文字集合登録の為に予約されている。
    • ESC k, ESC l, ESC m には昔は LS1R, LS2R, LS3R が与えられていたが、 現在は LS1R, LS2R, LS3R はそれぞれ ESC ~, ESC }, ESC | を用いる。 ただし、互換性の為に ESC k, ESC l, ESC m を認識する事もある。

  • CSI: Constrol Sequence Introducer

    CSI (constrol sequence introducer) で始まるシーケンスを特に CSI シーケンス (CSI sequence) または (狭義)制御シーケンス (constrol sequence) と呼ぶ。 プログラムに端末の特別な操作・状態・情報を送ったり (入力)、 プログラムから端末の状態を制御する命令を送ったり (出力) するのに用いられる。

    CSI は "ESC [" <x1B x5B> (または <x9B>) で表現される。 CSI は本来は C1 集合の <x9B> であるが、 UTF-8 を例として、文字コードによっては <0x9B> は通常の文字を表現するバイトである為(?)、 現在では <x9B> が用いられる事は少なく、専ら <x1B x5B> が用いられる。 今や <x9B> は CSI として解釈されない事も多い。 <I> は [\x20, \x30) の範囲の文字を指定し、 <Ft> は [\x40, \x7F) の範囲の文字を指定する。 <I>* <Ft> の組合せによって、その制御シーケンスが特にどの様な機能を持った物かを指定する。 それぞれの組合せに対応する制御シーケンスの機能は、端末に依存する所が大きい。 これについては、後でまとめる事にする。 <Fp> は [\x20, \x40) の範囲の文字を表す。 <Fp>* の部分は、 主に、文字 <Ft> で指定された機能に対する引数として働く。 多くの場合、/|([0-9]+;)*[0-9]+/ の形式を持つ。 場合によっては、更に異なる機能を指定するのに用いられる。 詳しくは後述する。

  • OSC: Operating System Command

    OSC (operating system command) で始まるシーケンスは、 OSC シーケンス (OSC sequence) または OS コマンド (operating system command) と呼ばれる。 OSC シーケンスは、主に端末エミュレータのウィンドウの制御 (出力)・状態報告 (入力) などに用いられる。

    OSC は "ESC ]" <x1B x5D> (または <x9D>) で表現される。 CSI と同様の事情で現在は <x1B x5D> が用いられる。 <N> は [0-9] の文字を表す。 <N>* はコマンドの種類を指定する番号である。 後に続く .* の部分は、コマンドの引数として与える文字列である。 BEL <x07> または ST <x1b x5c> (または <x9c>) はコマンドの終端を表す。 (従って、当然ながら文字列 .* には BEL 及び ST を含ませる事はできない。)

  • DCS: Device Control String

    DCS で始まるシーケンスは DCS シーケンス (DCS sequence) または Device Control String と呼ばれる。 DCS は "ESC P" <x1b x50> (または <x90>) で表現される。 CSI と同様に現在は <x1B x50> を用いる。 .* に内容を記述する。 ST <x1B x5C> (または <x9C>) はシーケンスの終端を表す。

  • SOS: Start of String

    SOS で始まるシーケンスは SOS シーケンス (SOS sequence) と呼ばれる。 SOS は "ESC X" <x1B x58> (または <x98>) で表現される。 CSI と同様に現在は <x1B x58> を用いる。 .* に内容を記述する。内容には SOS または ST を含まない。 ST <x1B x5C> (または <x9C>) はシーケンスの終端を表す。

  • PM: Privacy Message

    PM で始まるシーケンスは PM シーケンス (PM sequence) または Privacy Message と呼ばれる。 PM は "ESC ^" <x1B x5E> (または <x9E>) で表現される。 CSI と同様に現在は <x1B x5E> を用いる。 .* に内容を記述する。 ST <x1B x5C> (または <x9C>) はシーケンスの終端を表す。

  • APC: Application Program Command

    APC で始まるシーケンスは APC シーケンス (APC sequence) または Application Program Command と呼ばれる。 APC は "ESC _" <x1B x5F> (または <x9F>) で表現される。 CSI と同様に現在は <x1B x5F> を用いる。 .* に内容を記述する。 ST <x1B x5C> (または <x9C>) はコマンドの終端を表す。

記号

以降では説明の為に以下の記号を用いる

  • <I>: [\x20-\x2F] の範囲にある文字
  • <Fp>: [\x30-\x3F] の範囲にある文字
  • <Ft>: [\x40-\x7E] の範囲にある文字
  • <Ps>: [0-9] の文字の並びを表す。整数。
  • <Pm>: 複数の整数を ";" <x3B> で区切った物。

  • Mode ?1000 smSendMouse Send Mouse X & Y [既定値 off]

    端末側で発生したマウスボタンの押下・解放のイベント情報を送る。 マウスイベントの情報は以下のシーケンスを用いて送信される。

      • smExMouseSgr が有効になっている時には、smExMouseSgrR が用いられる。
      • smExMouseUrxvt が有効になっている時は、smExMouseUrxvtR が用いられる。
      • smExMouseUtf8 が有効になっている時は、smExMouseUtf8R が用いられる。

    • 二つ目の引数 X はマウスポインタの横位置を表す列番号である。端末の最も左の列が 1 に対応する。
    • 三つ目の引数 Y はマウスポインタの縦位置を表す行番号である。端末の一番上の行が 1 に対応する。
    • <C> は引数の整数値に 0x20 を加算した値をバイト値とする文字である。引数の値は最大 223 (= 0xFF-0x20) に制限される。
    • <C8> は引数の整数値に 0x20 を加算した値を文字コードと見て UTF-8 形式で符号化したバイト列である。
    • <Ps> は引数の整数値を整数リテラルで表現した /[0-9]+/ の形式の文字列である。
